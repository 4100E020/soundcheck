# SoundCheck 行動應用程式介面設計

## 設計理念

**Slogan**: 頻率對了,就一起去現場吧。

**定位**: 全台首個「音樂活動導向」的社交工具

**核心價值**: 透過「票根驗證」建立信任,透過「音樂數據」連結靈魂

## 設計原則

- **行動優先**: 所有設計基於手機直向模式 (9:16),單手操作友善
- **iOS 風格**: 遵循 Apple Human Interface Guidelines,打造原生感受
- **信任建立**: 透過票根驗證系統建立用戶間的信任機制
- **音樂連結**: 以音樂品味作為社交連結的核心

## 色彩系統

### 主色調
- **Primary (主色)**: `#FF6B35` (活力橘) - 代表音樂現場的熱情與活力
- **Secondary (輔色)**: `#6C63FF` (電音紫) - 代表音樂的節奏與律動

### 功能色
- **Success (成功)**: `#00D9A3` - 票根驗證通過
- **Warning (警告)**: `#FFC107` - 活動即將開始
- **Error (錯誤)**: `#FF5252` - 驗證失敗

### 中性色
- **Background (背景)**: Light `#FFFFFF` / Dark `#0A0A0A`
- **Surface (卡片)**: Light `#F8F9FA` / Dark `#1A1A1A`
- **Foreground (文字)**: Light `#1A1A1A` / Dark `#FFFFFF`
- **Muted (次要文字)**: Light `#6B7280` / Dark `#9CA3AF`
- **Border (邊框)**: Light `#E5E7EB` / Dark `#2D2D2D`

## 螢幕架構

### 底部導航欄 (4 個主要分頁)

1. **探索 (Discover)** - `house.fill` icon
   - 一般交友配對
   - 每日限量 30 人
   
2. **活動 (Events)** - `calendar` icon
   - 活動列表
   - 票務驗證
   - 揪團功能
   
3. **聊天 (Chat)** - `message.fill` icon
   - 私訊對話
   - 揪團群組
   
4. **我的 (Profile)** - `person.fill` icon
   - 音樂基因圖
   - 票夾
   - 設定

## 核心螢幕設計

### 1. 探索頁面 (Discover Screen)

**佈局**: 卡片式配對介面

**主要內容**:
- 用戶頭像 (依驗證狀態決定模糊度)
- 匹配度百分比 (基於音樂品味)
- 暱稱與年齡
- 共同喜愛的藝人 (Top 3)
- 即將參加的活動標籤

**互動方式**:
- 左滑: 跳過
- 右滑: 喜歡
- 點擊卡片: 查看詳細資料
- 特殊功能: 點歌破冰 (傳送一首適合的歌)

**限制機制**:
- 未驗證用戶: 每日限滑 30 人
- 已驗證用戶 (VVIP): 無限配對 + 可見「誰喜歡我」

### 2. 活動列表頁面 (Events List Screen)

**佈局**: 垂直滾動列表

**主要內容**:
- 活動封面圖 (高斯模糊背景)
- 活動名稱
- 日期與倒數計時
- 參加人數熱度
- 驗證狀態標記 (✅ VVIP / 🔒 未驗證)

**篩選功能**:
- 時間: 本週 / 本月 / 未來
- 類型: 音樂祭 / 演唱會 / Live House
- 地區: 北部 / 中部 / 南部

**排序方式**:
- 依日期 (預設)
- 依熱度
- 依匹配度 (有相同音樂品味的人數)

### 3. 活動詳情頁面 (Event Detail Screen)

**這是 App 最複雜且最重要的頁面**

#### Header 區域 (固定置頂)

**視覺設計**:
- 背景: 活動主視覺的高斯模糊圖
- 半透明遮罩確保文字可讀性

**資訊顯示**:
- 活動名稱 (粗體大字)
- 日期時間
- 倒數計時 (例: 還有 14 天)
- 地點

**CTA 按鈕**:
- 未驗證狀態: `[📷 上傳票根解鎖 VVIP]` (金色呼吸燈效果)
- 已驗證狀態: `[✅ VVIP 通行證]` + `[顯示我的入場 QR]`

#### Body 區域 (3 個分頁)

##### Tab 1: 情報 (Info)

**定位**: 資訊聚合中心

**內容區塊**:
1. **陣容 (Lineup)**
   - 依照用戶 Spotify 喜好排序
   - 優先顯示「你的菜」(匹配度高的藝人)
   - 每個藝人顯示: 頭像、名稱、匹配度標記

2. **活動熱度**
   - 參加人數統計 (例: 3,420 人參加)
   - 已驗證 VVIP 人數
   - 性別比例圖表

3. **官方資訊**
   - 購票連結 (外部連結)
   - 場地地圖 (整合 Google Maps)
   - 注意事項

**互動功能**:
- 加入行事曆
- 分享活動
- 設定提醒

##### Tab 2: 找人 (People)

**定位**: 1對1 靈魂配對

**佈局**: 雙欄卡片流 (類似 Instagram 探索頁)

**卡片內容**:
- 用戶頭像 (依驗證狀態決定模糊度)
- 匹配度百分比 (例: 92%)
- 狀態標籤 (例: 📍 找人喝酒、📷 互幫拍照)
- VVIP 徽章 (已驗證用戶)

**篩選選項**:
- 只看異性 / 同性
- 只看 VVIP (已驗證用戶)
- 依匹配度排序

**互動方式**:
- 右滑喜歡
- 點歌破冰 (傳送一首適合這活動的歌)
- 點擊查看詳細資料

**權限限制**:
- 未驗證: 每日限滑 30 人
- 已驗證: 無限配對 + 可見「誰喜歡我」

##### Tab 3: 揪團 (Crew)

**定位**: 多對多功能組隊

**佈局**: 列表式 (類似 PTT/Dcard)

**揪團類型標籤**:
- 🔴 [交通] - 共乘、拼車
- 🔵 [住宿] - 找室友、分攤房費
- 🟢 [現場] - 一起看演出、互拍照
- 🟡 [票券] - 票券交換、轉讓

**揪團卡片內容**:
- 標題 (例: "台北→高雄 3/28 早上出發")
- 類型標籤
- 發起人頭像與暱稱
- 人數狀態 (例: 缺 2 人 / 已滿團)
- 發布時間

**互動功能**:
- 發起揪團 (使用樣板精靈)
- 申請加入
- 收藏揪團

**權限限制**:
- 未驗證: 僅能瀏覽
- 已驗證: 可發文、可加入

### 4. 聊天頁面 (Chat Screen)

**佈局**: 對話列表

**主要內容**:
- 對話對象頭像
- 最後訊息預覽
- 時間戳記
- 未讀訊息數量
- 群組標記 (揪團群組)

**分類**:
- 全部
- 私訊
- 揪團群組

### 5. 個人資料頁面 (Profile Screen)

**佈局**: 垂直滾動

**主要區塊**:

1. **個人資訊**
   - 頭像
   - 暱稱
   - 年齡
   - VVIP 徽章 (已驗證用戶)

2. **音樂基因圖**
   - 視覺化圖表顯示音樂品味
   - Top 藝人列表
   - 音樂風格分布 (雷達圖)
   - 連結 Spotify 按鈕

3. **票夾**
   - 已驗證的票根列表
   - 即將參加的活動
   - 過往活動紀錄

4. **設定**
   - 帳號設定
   - 隱私設定
   - 通知設定
   - 關於 / 幫助

## 關鍵用戶流程

### 流程 1: 票根驗證

1. 用戶點擊活動詳情頁的「上傳票根解鎖 VVIP」按鈕
2. 開啟相機或相簿選擇票根圖片
3. 上傳圖片到伺服器
4. 顯示「驗證中」載入畫面
5. 後端 OCR 識別票根資訊
6. 驗證成功 → 顯示「✅ VVIP 通行證」+ 解鎖所有功能
7. 驗證失敗 → 顯示錯誤訊息與重試選項

### 流程 2: 配對與破冰

1. 用戶在探索頁面滑動卡片
2. 看到喜歡的對象,右滑或點擊「喜歡」
3. 選擇「點歌破冰」功能
4. 從推薦歌單中選擇一首歌
5. 附上簡短訊息 (選填)
6. 發送破冰訊息
7. 對方收到通知
8. 若對方也喜歡 → 配對成功,開啟聊天室

### 流程 3: 發起揪團

1. 用戶在活動詳情頁點擊「揪團」分頁
2. 點擊「發起揪團」按鈕
3. 選擇揪團類型 (交通/住宿/現場/票券)
4. 填寫樣板表單:
   - 交通: 出發地、目的地、時間、人數、費用分攤
   - 住宿: 地點、日期、房型、人數、費用分攤
   - 現場: 活動、時間、地點、人數、目的
   - 票券: 票種、數量、價格、交易方式
5. 填寫額外說明 (選填)
6. 發布揪團
7. 其他用戶可申請加入
8. 發起人審核申請
9. 成團後建立群組聊天室

## 特殊互動設計

### 票根驗證視覺回饋

- 上傳成功: 金色粒子動畫
- 驗證中: 脈衝式載入動畫
- 驗證成功: 綠色勾選動畫 + 觸覺回饋
- 驗證失敗: 紅色搖晃動畫 + 錯誤提示

### 配對成功動畫

- 雙方頭像從兩側飛入
- 中間爆出音符粒子
- 顯示「配對成功!」文字
- 觸覺回饋 (成功震動)

### 音樂基因圖視覺化

- 雷達圖顯示音樂風格分布
- 動態載入動畫
- 可與其他用戶比對
- 顯示相似度百分比

## 無障礙設計

- 所有互動元素至少 44x44 pt
- 充足的色彩對比度 (WCAG AA 標準)
- 支援動態字體大小
- VoiceOver 完整支援
- 觸覺回饋輔助視覺回饋

## 效能考量

- 圖片使用 WebP 格式並壓縮
- 列表使用 FlatList 虛擬化
- 圖片延遲載入
- 快取策略: 活動資料、用戶頭像
- 離線支援: 已載入的活動資訊可離線瀏覽
